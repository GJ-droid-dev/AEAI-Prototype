# Consortium Governance (Replacing the DAO)

## Overview

A Decentralized Autonomous Organization (DAO) governed by token holders introduces severe risks to an epistemic engine: truth should not be determined by those with the most capital.

Therefore, the protocol parameters, open-source codebase, and system architecture are governed by the **AEAI Consortium**—a multi-stakeholder standard body similar to the W3C, IETF, or the Linux Foundation.

## Structure of the Consortium

### 1. The Steering Committee
Composed of elected representatives from participating institutional nodes (universities, open-source foundations, investigative journalism networks). 
- **Mandate:** Vote on protocol upgrades, confidence formula weight adjustments, and epistemic tier reclassifications.
- **Voting Mechanism:** One Institution, One Vote. (No token-weighted voting).

### 2. The Technical Advisory Board
Composed of cryptography, AI safety, and epistemology experts.
- **Mandate:** Review all proposed changes to the Adversarial Validation Cycle (AVC) agents, Prompts, and LLM model architectures. Ensure no biases are introduced into the open-source logic.

### 3. Public Comment & Peer Review
Any proposed change to the network requires a mandatory public comment period (RFC style). Changes must undergo formal peer review before reaching the Steering Committee for a vote.

## Supermajority Requirements

To prevent hostile takeovers or sudden ideological shifts, changes require a **75% Supermajority Consensus**:
- Changes to the Epistemic Pyramid (Tier Weights).
- Upgrades to the Transparency Log architecture.
- Re-weighting of the Confidence Formula parameters.
- Axiom Promotions (Elevating a Tier 2/3 claim to a Tier 0/1 Axiom).

## The "No Kill Switch" Guarantee

The consortium governs the *code* and the *parameters*, but they do not hold administrative keys to the network. 
- There is no central server to shut down. 
- There is no single DNS to seize.
- There is no kill switch.

Even if the Consortium is entirely compromised by a state actor, they cannot rewrite past verdicts stored in the Merkle Transparency Logs. They can only propose malicious code updates, which the independent node operators would simply refuse to install, resulting in a hard fork that preserves the original, uncorrupted truth engine.
